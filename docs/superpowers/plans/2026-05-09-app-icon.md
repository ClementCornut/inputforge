# App Icon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the dx-cli rocket placeholder and the unprovenanced 4096-byte `icon.rgba` with a branded mark, generated from three committed SVGs by a single bun script, embedded into `inputforge-app.exe` via `winres`, and declared for `dx bundle` via a new `Dioxus.toml`.

**Architecture:** Three layers, each independently testable.

1. **Sources (committed, human-edited):** three SVGs at `crates/inputforge-app/assets/icon{,-16,-24}.svg` plus a `README.md` that explains the contract.
2. **Pipeline (committed, deterministic):** `scripts/build-app-icon.mjs` rasterises the SVGs through `sharp` and packs the multi-resolution `.ico` through `to-ico`, emitting five PNGs, one ICO, and one 32x32 RGBA blob. Re-running on a clean tree leaves `git status` clean.
3. **Wiring (compile-time, contingency-aware):** `winres` in `crates/inputforge-app/build.rs` embeds `icon.ico` into the PE icon resource so taskbar, alt-tab, Explorer, and shortcut surfaces all read the same mark. A separate `Dioxus.toml` declares the same assets for `dx bundle`. If dx-cli's auto-injected rocket overrides the winres resource at `dx run` time, the pre-flight check in Task 6 detects this before any winres commit lands, and `Dioxus.toml`'s `[application].icon` carries the dev-time icon instead.

The order matters: SVG sources → script → generated binaries → README → pre-flight winres survival check (working tree only) → conditional winres commits (only if pre-flight passes) → Dioxus.toml bundle config → final acceptance.

**Tech Stack:** Bun 1.x runtime, `sharp ^0.33` (SVG to PNG and raw RGBA), `to-ico ^1.1` (multi-resolution ICO packer), Rust 2024, `winres 0.1` (Windows resource embedding via build.rs), Dioxus 0.7 / dx-cli 0.7.6 (bundle config).

## Sequencing

The plan runs in four phases:

- **Phase 1 (Tasks 1-5): Source-of-truth assets.** SVGs, npm setup, script, generated binaries, README. Each commit leaves the repo in a runnable state because nothing yet consumes the new files.
- **Phase 2 (Task 6): Pre-flight winres survival check.** All three winres edits applied to the working tree without committing; verified via `cargo build`, `dx build`, and `dx run`. Decides which path Phase 3 takes. This implements spec verification 0 literally: no winres plumbing reaches main until it has been confirmed to survive dx-cli.
- **Phase 3 (Tasks 7-8, CONDITIONAL on Task 6 outcome): Commit the winres setup.** Runs only if Task 6 confirms winres survives `dx run`. If Task 6 finds dx-cli overwrites the embed, both tasks are skipped (Task 6 ends by discarding the working-tree changes) and Phase 4's `Dioxus.toml` carries the dev-time icon via `[application].icon` instead.
- **Phase 4 (Tasks 9-10): Bundle config and final acceptance.** Creates `Dioxus.toml` (with or without `[application].icon` per the Task 6 decision) and walks the spec verification list end to end.

`cargo build -p inputforge-app` stays green at every commit boundary in phases 1, 3, and 4. (Phase 2 is working-tree only and produces no commits.)

### Task 3: Write the icon-generation script

**Files:**
- Create: `scripts/build-app-icon.mjs`

- [ ] **Step 1: Create the script**

Create `scripts/build-app-icon.mjs` with this content. The script implements the pipeline from the spec section "Generation Script": render PNGs at 16/24/32/48/64/256 (48 only as an ICO frame, not as a standalone PNG), pack the ICO, render the 32x32 RGBA, and verify the dot-pixel sanity check before writing.

If `sharp` or its bundled libvips changes major version, the byte-determinism contract resets: regenerate via Task 4 and recommit any binary deltas. Major-version bumps to either dependency are regen events.

```js
#!/usr/bin/env bun
// Generate app-icon binary outputs from the SVG sources.
// Source contract: crates/inputforge-app/assets/README.md.
// Idempotent: for a pinned sharp + libvips, same SVGs in produce
// byte-identical files out. Bumping sharp's major version is a regen event.

import { readFile, writeFile } from "node:fs/promises";
import { fileURLToPath } from "node:url";
import { dirname, join } from "node:path";
import sharp from "sharp";
import toIco from "to-ico";

const scriptDir = dirname(fileURLToPath(import.meta.url));
const repoRoot = join(scriptDir, "..");
const assetsDir = join(repoRoot, "crates", "inputforge-app", "assets");

const sources = {
  master: join(assetsDir, "icon.svg"),
  small16: join(assetsDir, "icon-16.svg"),
  medium24: join(assetsDir, "icon-24.svg"),
};

async function renderPng(svgPath, size) {
  const svg = await readFile(svgPath);
  return sharp(svg).resize(size, size).png().toBuffer();
}

async function renderRgba(svgPath, size) {
  const svg = await readFile(svgPath);
  return sharp(svg).resize(size, size).ensureAlpha().raw().toBuffer();
}

async function main() {
  const masterAt32 = await renderPng(sources.master, 32);
  const masterAt48 = await renderPng(sources.master, 48);
  const masterAt64 = await renderPng(sources.master, 64);
  const masterAt256 = await renderPng(sources.master, 256);
  const small16Buf = await renderPng(sources.small16, 16);
  const medium24Buf = await renderPng(sources.medium24, 24);

  const pngOutputs = [
    ["icon-16.png", small16Buf],
    ["icon-24.png", medium24Buf],
    ["icon-32.png", masterAt32],
    ["icon-64.png", masterAt64],
    ["icon-256.png", masterAt256],
  ];
  for (const [name, buf] of pngOutputs) {
    await writeFile(join(assetsDir, name), buf);
  }

  // ICO entry order: 16, 32, 48, 256. to-ico packs in argument order;
  // Windows reads ICONDIRENTRY records by id, not order, so this is
  // ergonomic for inspection rather than functional.
  const icoBuf = await toIco([small16Buf, masterAt32, masterAt48, masterAt256]);
  await writeFile(join(assetsDir, "icon.ico"), icoBuf);

  // 32x32 raw RGBA for tray-icon's Icon::from_rgba. ensureAlpha() guarantees
  // the alpha plane is present even when sharp's RGB-only fast path triggers.
  const rgba = await renderRgba(sources.master, 32);
  if (rgba.length !== 32 * 32 * 4) {
    throw new Error(
      `RGBA buffer is ${rgba.length} bytes; expected ${32 * 32 * 4}`,
    );
  }

  // Dot-pixel sanity check at (16, 16). The dot is r=2 in 32x32 raster,
  // so the centre pixel sits inside the disc and reads as solid CRT
  // Phosphor Green at full alpha.
  const idx = (16 * 32 + 16) * 4;
  const got = [rgba[idx], rgba[idx + 1], rgba[idx + 2], rgba[idx + 3]];
  const want = [46, 224, 160, 255];
  const tolerance = 4;
  for (let c = 0; c < 4; c += 1) {
    if (Math.abs(got[c] - want[c]) > tolerance) {
      throw new Error(
        `RGBA dot pixel sanity check failed: got (${got.join(", ")}), expected ~(${want.join(", ")})`,
      );
    }
  }
  await writeFile(join(assetsDir, "icon.rgba"), rgba);

  const summary = [
    ...pngOutputs.map(([name, buf]) => [name, buf.length]),
    ["icon.ico", icoBuf.length],
    ["icon.rgba", rgba.length],
  ];
  console.log("Generated icon assets:");
  for (const [name, size] of summary) {
    console.log(`  ${name}  ${size} bytes`);
  }
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

The three SVG files in this directory are the source of truth for the app

bun scripts/build-app-icon.mjs

| ------------- | ----------------- | --------------------------------------------- |

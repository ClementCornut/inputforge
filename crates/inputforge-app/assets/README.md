# App Icon Assets

The three SVG files in this directory are the source of truth for the app
icon. The PNGs, ICO, and RGBA blob are generated outputs, regenerated by:

```
bun scripts/build-app-icon.ts
```

(Run from the workspace root. Requires `bun install` to have been run once.)

## Inputs and outputs

| Source        | Sizes rendered    | Outputs                                       |
| ------------- | ----------------- | --------------------------------------------- |
| `icon.svg`    | 32, 48, 64, 256   | `icon-32.png`, `icon-64.png`, `icon-256.png`, ICO frames at 32/48/256, `icon.rgba` (32x32) |
| `icon-16.svg` | 16                | `icon-16.png`, ICO frame at 16                |
| `icon-24.svg` | 24                | `icon-24.png` (bundle only; 24 has no ICO frame) |

The ICO carries 16, 32, 48, 256. The bundle PNG set is 16, 24, 32, 64, 256
(the sizes the Dioxus bundle config accepts). The 48 px frame lives only
inside the ICO.

## Rule

Do not edit the PNGs, ICO, or RGBA by hand. Edit the SVGs and re-run the
script. The committed binaries are the contract for users without bun
installed; bun is only required to regenerate them.

## Why three SVGs

The 96 px master uses a 9 px stroke and r=6 dot, both of which collapse to
sub-pixel widths if scaled to 16 or 24 directly. The 16 px and 24 px
variants are hand-tuned for their raster grids (corner radius, stroke
weight, dot radius) so the mark stays legible at the smallest taskbar
sizes. See `docs/superpowers/specs/2026-05-09-app-icon-design.md` for the
brand-rule rationale.
